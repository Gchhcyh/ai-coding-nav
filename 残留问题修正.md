## 1. 🔴 Sitemap 写入路径问题（必须修）

**问题**：`npm run build` 执行顺序是 `next build` → `node scripts/generate-sitemap.js`。其中 `next build` 会将 `public/` **拷贝到** `out/`，然后脚本才往 `public/sitemap.xml` 写新内容。结果是 Cloudflare Pages 部署时用的是 `out/` 里的旧 sitemap（来自 git 仓库版本，而非脚本生成的版本）。

**文件**：`scripts/generate-sitemap.js`

**修复**：在文件末尾（`fs.writeFileSync(...)` 之后）追加：

```js
// Also write to out/ so Cloudflare Pages deploys the latest sitemap
const outDir = path.join(__dirname, '..', 'out');
if (fs.existsSync(outDir)) {
  fs.writeFileSync(path.join(outDir, 'sitemap.xml'), sitemap, 'utf-8');
  console.log(`Also written to out/sitemap.xml`);
}
```

改完后 `generate-sitemap.js` 完整内容：

```js
const fs = require('fs');
const path = require('path');

const BASE_URL = 'https://ai-coding-nav.pages.dev';
const tools = JSON.parse(fs.readFileSync(path.join(__dirname, '..', 'data', 'tools.json'), 'utf-8'));

const urls = [
  { loc: `${BASE_URL}/`, priority: '1.00' },
  ...tools.map(t => ({
    loc: `${BASE_URL}/tools/${t.slug}`,
    priority: t.featured ? '0.90' : '0.80',
  })),
];

const sitemap = `<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
${urls.map(u => `  <url>
    <loc>${u.loc}</loc>
    <priority>${u.priority}</priority>
  </url>`).join('\n')}
</urlset>
`;

fs.writeFileSync(path.join(__dirname, '..', 'public', 'sitemap.xml'), sitemap, 'utf-8');
console.log(`Sitemap generated: ${urls.length} URLs`);

// Also write to out/ so Cloudflare Pages deploys the latest sitemap
const outDir = path.join(__dirname, '..', 'out');
if (fs.existsSync(outDir)) {
  fs.writeFileSync(path.join(outDir, 'sitemap.xml'), sitemap, 'utf-8');
  console.log(`Also written to out/sitemap.xml`);
}
```

## 2. 🟡 Newsletter JSONP 缺少超时处理（强烈建议）

**问题**：JSONP 方式如果 Mailchimp 服务器不响应，`script.onerror` 不一定触发，用户会永远卡在 "提交中..." 状态。

**文件**：`components/Newsletter.tsx`

**修复**：在 `handleSubmit` 函数内，`const callbackName` 之后、`window[callbackName]` 之前加入 timeout，在回调和 onerror 中清除。

具体改动位置如下：

```tsx
const callbackName = `mailchimp_cb_${Date.now()}`;
const url = new URL("https://dev.us11.list-manage.com/subscribe/post-json");
url.searchParams.set("u", "288932316a3d3fd2669581bbe");
url.searchParams.set("id", "993f79b613");
url.searchParams.set("EMAIL", email);
url.searchParams.set("c", callbackName);

// ===== 新增 START =====
const timeout = setTimeout(() => {
  if (window[callbackName]) {
    delete window[callbackName];
    document.head.removeChild(script);
    setStatus("error");
    setMessage("请求超时，请稍后重试");
  }
}, 10000);
// ===== 新增 END =====

window[callbackName] = (data: { result: string; msg: string }) => {
  clearTimeout(timeout);  // ← 新增这一行
  delete window[callbackName];
  document.head.removeChild(script);
  if (data.result === "success") {
    setStatus("success");
    setMessage(data.msg || "确认邮件已发送，请检查收件箱");
    setEmail("");
  } else {
    setStatus("error");
    setMessage(data.msg || "订阅失败，请检查邮箱地址");
  }
};

const script = document.createElement("script");
script.src = url.toString();
script.onerror = () => {
  clearTimeout(timeout);  // ← 新增这一行
  delete window[callbackName];
  if (script.parentNode) document.head.removeChild(script);
  setStatus("error");
  setMessage("网络错误，请稍后重试");
};
document.head.appendChild(script);
```

## 3. 🟢 Footer Twitter/X 链接仍是占位（建议上线前处理）

**文件**：`components/Footer.tsx` 第 13 行附近

**现状**：
```tsx
<a href="https://x.com" ...>Twitter/X</a>
```

指向 `x.com` 首页而非项目账号。Product Hunt 上线时建议有一个真实账号。

**方案 A** — 已有账号：替换为实际链接：
```tsx
<a href="https://x.com/你的账号" ...>Twitter/X</a>
```

**方案 B** — 暂无账号：先隐藏或替换为 GitHub：
```tsx
<a href="https://github.com/Gchhcyh/ai-coding-nav" ...>GitHub</a>
```
