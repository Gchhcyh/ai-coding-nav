### 1. Sitemap 只有首页，78 个工具页全未收录

**文件**：`public/sitemap.xml`

**现状**：只有 1 个 `<url>`（首页），Google 无法发现工具详情页。

**修复**：在 `next build` 后、部署前，重新生成 sitemap。添加一个 `scripts/generate-sitemap.js`：

```js
// scripts/generate-sitemap.js
const fs = require("fs");
const path = require("path");
const tools = require("../data/tools.json");

const BASE = "https://ai-coding-nav.pages.dev";
const LAST_MOD = new Date().toISOString().split("T")[0];

const urls = [
  { loc: `${BASE}/`, priority: "1.0" },
  ...tools.map((t) => ({
    loc: `${BASE}/tools/${t.slug}/`,
    priority: "0.8",
  })),
];

const xml = `<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
${urls
  .map(
    (u) => `  <url>
    <loc>${u.loc}</loc>
    <lastmod>${LAST_MOD}</lastmod>
    <changefreq>weekly</changefreq>
    <priority>${u.priority}</priority>
  </url>`
  )
  .join("\n")}
</urlset>
`;

fs.writeFileSync(path.join(__dirname, "../public/sitemap.xml"), xml);
console.log(`Sitemap generated: ${urls.length} URLs`);
```

然后在 `package.json` 加一个脚本：

```json
"scripts": {
  "build": "next build && node scripts/generate-sitemap.js"
}
```

**验证**：构建后检查 `out/sitemap.xml` 包含 79 个 URL。

### 3. 分类筛选不更新 URL（state 和 URL 不同步）

**文件**：`app/page.tsx`

**现状**：
- Header 分类链接 → 改 URL（`/?category=xxx`）→ `useEffect` 同步 state ✅
- CategoryFilter 按钮 → 只调 `setActiveCategory`，不改 URL ❌

**后果**：用户点分类标签后，URL 不更新，刷新页面/分享链接状态丢失。

**修复**：在 HomePage 组件中，使用 `useRouter` 同步 URL：

```tsx
// app/page.tsx — 在 HomePage 组件顶部添加
import { useRouter } from "next/navigation";

function HomePage() {
  const searchParams = useSearchParams();
  const router = useRouter();  // 新增
  // ...

  // 修改 CategoryFilter 的 onChange 调用方式
  const handleCategoryChange = (category: string) => {
    setActiveCategory(category);
    const params = new URLSearchParams(searchParams.toString());
    if (category) {
      params.set("category", category);
    } else {
      params.delete("category");
    }
    const query = params.toString();
    router.push(query ? `/?${query}` : "/", { scroll: false });
  };
```

然后把 `<CategoryFilter active={activeCategory} onChange={setActiveCategory} />` 改为：

```tsx
<CategoryFilter active={activeCategory} onChange={handleCategoryChange} />
```

### 7. Newsletter 订阅成功判断不准确

**文件**：`components/Newsletter.tsx`，第 27-30 行

**问题**：`fetch(url, { mode: "no-cors" })` 拿不到响应体，只要没抛网络异常就显示"订阅成功"。Mailchimp 返回错误（邮箱无效/已订阅）也会误报成功。

**修复方案**：改用真正的 JSONP 方式调用 Mailchimp。

```tsx
const handleSubmit = useCallback(async (e: React.FormEvent) => {
  e.preventDefault();
  if (!email) return;

  setStatus("loading");
  setMessage("");

  const callbackName = `mailchimp_cb_${Date.now()}`;

  return new Promise<void>((resolve) => {
    const script = document.createElement("script");
    const timeout = setTimeout(() => {
      cleanup();
      setStatus("error");
      setMessage("请求超时，请稍后重试");
      resolve();
    }, 10000);

    const cleanup = () => {
      clearTimeout(timeout);
      delete (window as any)[callbackName];
      if (script.parentNode) script.parentNode.removeChild(script);
    };

    (window as any)[callbackName] = (data: { result: string; msg: string }) => {
      cleanup();
      if (data.result === "success") {
        setStatus("success");
        setMessage("确认邮件已发送，请检查收件箱");
        setEmail("");
      } else {
        setStatus("error");
        // 提取 Mailchimp 返回的友好提示
        const msg = data.msg?.replace(/<[^>]*>/g, "").split(" - ")[0] || "订阅失败";
        setMessage(msg);
      }
      resolve();
    };

    const url = `https://dev.us11.list-manage.com/subscribe/post-json?u=288932316a3d3fd2669581bbe&id=993f79b613&EMAIL=${encodeURIComponent(email)}&c=${callbackName}`;
    script.src = url;
    script.onerror = () => {
      cleanup();
      setStatus("error");
      setMessage("网络错误，请稍后重试");
      resolve();
    };
    document.head.appendChild(script);
  });
}, [email]);
```

### 9. GA 事件追踪不完整（缺出站点击）

**文件**：`app/layout.tsx`（添加全局事件处理） + `components/ToolCard.tsx`

**对于导航站最关键的两个事件**：

**(A) 工具出站点击** — 知道哪些工具被点击最多，直接影响广告定价。

修改 `components/ToolCard.tsx`，在 `Link` 的 `href` 前加一层拦截：

```tsx
// ToolCard 组件中
function trackOutbound(toolName: string, toolUrl: string) {
  if (typeof window !== "undefined" && (window as any).gtag) {
    (window as any).gtag("event", "tool_click", {
      event_category: "outbound",
      event_label: toolName,
      value: toolUrl,
    });
  }
}

// 非对比模式下的返回 JSX：
return (
  <Link
    href={tool.url}
    onClick={() => trackOutbound(tool.name, tool.url)}
    // ...
  >
    {cardContent}
  </Link>
);
```

**(B) 搜索事件** — 知道用户在搜什么。

修改 `components/SearchBar.tsx`，在搜索触发时上报：

```tsx
useEffect(() => {

    (window as any).gtag("event", "search", {
      event_category: "engagement",
      event_label: query.trim(),
    });
  }
```

### 11. 构建后工具页数量验证

**文件**：`package.json` — 在 build 脚本后加验证步骤。

**问题**：目前没有任何自动化检查，万一 `out/` 里缺页面，部署了才发现。

**修复**：在 `scripts/` 下加一个验证脚本，build 后自动执行：

```js
// scripts/verify-build.js
const fs = require("fs");
const path = require("path");
const tools = require("../data/tools.json");

let errors = [];

// 检查每个工具页是否存在
for (const tool of tools) {
  const htmlPath = path.join(__dirname, "../out/tools", `${tool.slug}.html`);
  if (!fs.existsSync(htmlPath)) {
    errors.push(`Missing: ${tool.slug}`);
  }
}

// 检查首页
if (!fs.existsSync(path.join(__dirname, "../out/index.html"))) {
  errors.push("Missing: index.html");
}

// 检查 sitemap 行数
const sitemap = fs.readFileSync(path.join(__dirname, "../out/sitemap.xml"), "utf8");
const urlCount = (sitemap.match(/<url>/g) || []).length;
if (urlCount < tools.length + 1) {
  errors.push(`Sitemap only has ${urlCount} URLs, expected ${tools.length + 1}+`);
}

if (errors.length > 0) {
  console.error("❌ Build verification FAILED:");
  errors.forEach((e) => console.error("  - " + e));
  process.exit(1);
}
console.log(`✅ Build OK: ${tools.length + 1} pages, sitemap ${urlCount} URLs`);
```

`package.json` build 脚本：

```json
"scripts": {
  "dev": "next dev",
  "build": "next build && node scripts/generate-sitemap.js && node scripts/verify-build.js",
  "start": "next start",
  "lint": "next lint"
}
```
