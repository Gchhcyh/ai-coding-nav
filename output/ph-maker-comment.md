# Maker Comment - AI Coding Nav

**作者：Marvis（AI Coding Nav 创建者）**

---

我们的 Coming Soon 页面挂了整整两周，猜猜多少人来瞄过？3 个。其中两个是我自己登出后查看的，第三个可能是我妈。

所以当昨天正式上线时，我已经做好了"没人来"的心理准备。但说实话，我其实没那么在意——因为这个项目本身就不是冲着"爆红"做的。

## 为什么做这个站？因为我自己快被逼疯了

过去三个月，我每天都在选 AI 编程工具。今天 Cursor 出了新功能，明天 Claude Code 说碾压一切，后天某篇博客说"GPT-5 会让所有 IDE 插件失业"。我翻了十几个工具目录站，想找个靠谱的对比——结果发现它们全都一个样：一长串工具名 + 一句话简介 + 一个链接。没了。

这就好比你去餐厅点菜，服务员给了你一本菜单，上面只有菜名和"好吃"两个字。没有配料表、没有口味说明、没人告诉你这道菜和那道菜的区别。

然后你点了一道菜，难吃。服务员说："哦，那道菜其实不适合你这种口味。"

**那你为什么不早说？？？**

## AI Coding Nav 不是又一份 "工具清单"

市面上 90% 的 AI 工具目录对开发者来说没用——它们只是披着"导航"外皮的 SEO 农场。搜索引擎上搜"AI 开发工具推荐"，前两页全是相同内容的镜像站，用 AI 生成一堆重复废话，挂满广告，就是为了赚点可怜的 AdSense。

这也是我决定做一个**真正有态度的目录**的原因。AI Coding Nav 的核心不是"收录最多"，而是：

- **对比模式**：最多 4 个工具并排对比，看看到底谁强谁弱
- **真实笔记**：每个工具的实测体验，包括优点和坑——不是那种"功能强大、体验流畅"的废话
- **场景推荐**：不确定该选哪个？告诉网站你的需求，它会帮你匹配
- **决策辅助**：每个工具都有明确的"适合谁"和"不适合谁"

我花了整整两周，测了 18 个工具，最终选了 3 个放进自己的工作流。笔记区里写的就是这个过程——不加滤镜，有好说好，有坑说坑。

## 一个争议观点

**90% 的 AI 编程工具目录对开发者没用——它们只是 SEO 农场。**

这些站的问题不在于内容少，而在于它们根本不想解决"选择困难"这个问题。因为"选择困难"一旦解决，用户就走了，广告费就没了。它们的商业模式决定了它们需要你迷茫，需要你在列表里反复滚动，多刷几个广告。

我理解这可能是"站着说话不腰疼"——我的站目前一个广告都没有（虽然 GitHub README 里列了广告位规划，但那是未来的事，而且只打工具方的广告，不给垃圾广告联盟打工）。

如果你觉得这个观点太极端，我接受反驳。但我更想听到的是：**你在选 AI 工具时，最困扰的问题是什么？** 来评论区告诉我，我会在笔记区直接写一篇回应。
